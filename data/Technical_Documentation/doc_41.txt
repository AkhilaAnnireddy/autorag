Monitoring and Alerting Setup Guide
Introduction
The purpose of this guide is to provide a comprehensive overview of the monitoring and alerting setup for the IT infrastructure of XYZ Corporation. The guide is intended for system administrators and IT staff who are responsible for ensuring the smooth operation of the company's IT systems.

Monitoring Tools
The company uses a combination of monitoring tools to ensure the IT infrastructure is running smoothly. These tools include Nagios, Prometheus, and Grafana. The following table provides a summary of the tools and their functions.

Table 2: Monitoring Tools
| Tool | Function | Frequency |
| --- | --- | --- |
| Nagios | System and network monitoring | Every 5 minutes |
| Prometheus | System and application monitoring | Every 1 minute |
| Grafana | Data visualization and alerting | On demand |

Alerting Policies
The company has established alerting policies to ensure that IT staff are notified in a timely manner in case of any issues. The following table provides a summary of the alerting policies.

Table 3: Alerting Policies
| Policy | Description | Notification |
| --- | --- | --- |
| System Down | Notification when a system is down | Email and SMS to IT staff |
| High CPU Usage | Notification when CPU usage exceeds 80% | Email to IT staff |
| Disk Space Low | Notification when disk space falls below 10% | Email to IT staff |
