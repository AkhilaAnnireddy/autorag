Containerization with Docker is a lightweight and portable way to deploy applications. It allows developers to package their code, along with its dependencies and libraries, into a single container that can be run on any system that supports Docker, without requiring a specific environment to be set up. Docker was first released in 2013 by Docker Inc., a company founded by Solomon Hykes, and has since become one of the most popular containerization platforms. The first version of Docker, 0.1, was released on March 13, 2013, and the latest version, 20.10.7, was released on May 4, 2021. Docker containers are much lighter than virtual machines, as they do not require a separate operating system for each container, but instead share the host's operating system. This makes them more efficient in terms of resources and faster to spin up and down. The process of containerizing an application with Docker involves several steps, including creating a Dockerfile, building a Docker image, and running a Docker container. A Dockerfile is a text file that contains instructions for building a Docker image, and it is used to define the base image, copy files, install dependencies, and set environment variables. For example, a Dockerfile for a Python application might include the following instructions: 
1. FROM python:3.9-slim
2. WORKDIR /app
3. COPY requirements.txt .
4. RUN pip install -r requirements.txt
5. COPY . .
6. CMD ["python", "app.py"] 
The Dockerfile is then used to build a Docker image, which is a binary package that contains the application code and its dependencies. The Docker image is built using the Docker build command, and it can be pushed to a Docker registry such as Docker Hub. Once the Docker image is built and pushed to a registry, it can be pulled and run on any system that supports Docker. 
Here is an example of how to build and run a Docker container: 
| Command | Description |
| --- | --- |
| docker build -t my-app . | Build a Docker image with the tag my-app |
| docker run -p 8080:8080 my-app | Run a Docker container from the my-app image and map port 8080 on the host to port 8080 in the container |
| docker ps | List all running Docker containers |
| docker stop my-container | Stop a running Docker container with the name my-container |
| docker rm my-container | Remove a stopped Docker container with the name my-container
