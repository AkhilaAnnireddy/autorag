
   <h1>Database Schema Documentation</h1>
   <h2>Introduction</h2>
   <p>The database schema for the fictional company, GreenTech Inc., was designed and implemented by the IT department in collaboration with the software development team. The database is used to store information about employees, customers, orders, and products. The schema was created using MySQL Workbench and is compatible with MySQL version 8.0. The database is hosted on a Linux server with a 2.5 GHz quad-core processor and 16 GB of RAM.</p>
   <h2>Entity Relationship Diagram</h2>
   <p>The entity relationship diagram (ERD) for the GreenTech Inc. database is shown in Table 1.</p>
   <table>
      <tr>

      </tr>
      <tr>

         <td>one-to-many with Order</td>
      </tr>
      <tr>
         <td>Customer</td>
         <td>customer_id, name, email, address</td>
         <td>one-to-many with Order</td>
      </tr>
      <tr>
         <td>Order</td>
         <td>order_id, order_date, total, employee_id, customer_id</td>

      <tr>
         <td>Product</td>
         <td>product_id, name, price, description</td>

   <h2>Indexing and Constraints</h2>
   <p>The database uses indexing and constraints to improve query performance and maintain data integrity. The primary keys for each table are employee_id, customer_id, order_id, and product_id. Foreign keys are used to establish relationships between tables.</p>
   <h2>Security and Backup</h2>
   <p>The database is secured using a combination of username and password authentication, as well as SSL encryption. Regular backups are performed daily at 2:00 AM using the mysqldump command. The backups are stored on an external hard drive and retained for 30 days.</p>
   <h2>Conclusion</h2>
   <p>The GreenTech Inc. database schema is designed to support the company's business operations and provide a scalable and secure platform for data storage and retrieval. The database is regularly maintained and updated to ensure optimal performance and data integrity.</p>
